# Чего сейчас не хватает для полноценной онлайн‑школы (пункты)

## 1) Учебный контур (ядро LMS)
- **Курсы/программы обучения**: структура “курс → модули → уроки”, цели, прогресс по модулям.
- **Учебный план и траектории**: индивидуальные планы, контроль дедлайнов, темы “пройдены/в работе/провалены”.
- **Единая система оценивания**: баллы/шкалы/критерии, итоговые оценки, журнал, средние по темам.

## 2) Домашка/проверка (workflow)
- **Единый журнал проверок** для преподавателя: очередь “что проверить”, приоритеты, дедлайны, фильтры по классу/ученику/типу работы.
- **Рубрики/критерии** (особенно для 24–27): шаблоны проверки, чек-листы, типовые ошибки.
- **Глобальная проверка по уроку/работе**: итоговый комментарий + итоговая отметка + авто‑пересчёт статистики.
- **Повторные попытки/пересдачи** как сущность (а не только “returned”).

## 3) Контент и библиотека
- **Библиотека материалов**: хранение, теги, поиск, версии, переиспользование между уроками.
- **Шаблоны уроков/комнат**: типовые структуры, чтобы уроки создавались “как продукт”.
- **Конструктор контента** (кроме Markdown): блоки, изображения, задания встраиваемые, раздатки.

## 4) Коммуникации
- **Чат/диалоги** (ученик↔препод/куратор/родитель) + привязка к уроку/работе.
- **Уведомления** (внутренние + Telegram/email): дедлайны, “вернули на доработку”, новый комментарий, назначена работа.
- **Календарь занятий для всех ролей**: подтверждение/перенос/отмена с прозрачной историей.

## 5) Управление пользователями (операционка школы)
- **Группы/классы** как сущность (не только связи “тьютор‑ученик”): расписание, массовые действия, отчёты по группе.
- **Роли уровня школы**: куратор/методист/проверяющий, разграничение доступа по предметам/классам.
- **Онбординг**: приглашения, подтверждения, восстановление доступа, контроль заполненности профилей.

## 6) Аналитика и качество обучения
- **Дашборды для ролей**: ученик (прогресс/план), родитель (динамика/риски), преподаватель (группа/очередь/результаты).
- **Система диагностик**: входной тест, контрольные точки, “дыры” по темам, персональные рекомендации.
- **Отчёты и экспорт**: в PDF/таблицы, история успеваемости, посещаемость, журнал.

## 7) Биллинг и юридический слой (если это коммерческая школа)
- **Тарифы/оплаты/подписки**, статусы оплат, доступы по оплате.
- **Договор/оферта**, согласия, хранение политики/логирование согласий.

## 8) Надёжность и продуктовые мелочи
- **Единая система загрузок** (файлы, лимиты, типы, защита ссылок).
- **Единая навигация и консистентный UI** на всех страницах.
- **Системные статусы и аудит действий** для спорных ситуаций (кто и когда изменил оценку/комментарий/статус).
